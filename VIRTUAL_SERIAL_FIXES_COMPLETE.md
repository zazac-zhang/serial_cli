## ⚠️ 已知限制

### 1. 监控功能限制
```bash
⚠️ Virtual port monitoring is limited.
   The sniffer is created but won't capture actual bridge traffic.
   For full monitoring, use regular serial ports.
```

**原因**: 实现完整的桥接流量监控需要大幅增加复杂度

**建议**: 使用真实串口进行完整监控

---

### 2. 进程局部注册表
```bash
# ⚠️ 必须在同一终端中管理虚拟串口

# 正确用法:
# 终端 1: 创建和管理
serial-cli virtual create
serial-cli virtual list
serial-cli virtual stop <id>

# 终端 2 和 3: 使用串口
serial-cli interactive --port /dev/ttys014
```

**原因**: 使用进程内 HashMap 存储虚拟串口对

**未来改进**: 可实现基于文件的注册表

---

### 3. 轮询 vs 异步 I/O
当前使用 1ms 轮询而非真正的异步 I/O

**当前性能**:
- 延迟: ~1ms (比之前的 10ms 好得多)
- CPU: 轻微增加
- 吞吐量: 对大多数用例足够

**未来改进**: 实现真正的 Tokio 异步 I/O 可进一步降低延迟
